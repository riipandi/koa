# Package Manager

Koa has **built-in package manager** for dependency management (like Zig `zig fetch` and Go `go get`).

## Philosophy

- **Built-in** - Package manager integrated in compiler
- **Git-based** - Dependencies from Git repositories
- **No registry** - No central package registry
- **Lockfile** - `Koa.lock` for reproducible builds
- **Simple** - Minimal configuration, convention over convention

---

## Koa.toml

The `Koa.toml` file in the project root:

```toml
[package]
name = "myproject"
version = "0.1.0"
type = "executable"  # or "library"

[dependencies]
# Git repository
http = { git = "https://github.com/riipandi/koa-http", version = "0.1.0" }
json = { git = "https://github.com/riipandi/koa-json", branch = "main" }

# Local path
utils = { path = "./utils" }

# Standard library (virtual)
std = { version = "0.1.0" }  # Always available
```

## Koa.lock

Generated lockfile:

```toml
# This file is automatically generated
# DO NOT EDIT

[[package]]
name = "http"
version = "0.1.0"
git = "https://github.com/riipandi/koa-http"
rev = "a1b2c3d4e5f6..."
checksum = "sha256:..."

[[package]]
name = "json"
version = "0.1.0"
git = "https://github.com/riipandi/koa-json"
branch = "main"
rev = "f6e5d4c3b2a1..."
checksum = "sha256:..."

[[package]]
name = "utils"
version = "0.1.0"
path = "./utils"
checksum = "sha256:..."
```

## Commands

All package management commands are under the `pkg` subcommand:

### koa pkg fetch

Download dependencies:

```bash
# Download all dependencies
koa pkg fetch

# Output:
# в Ӣ Fetching dependencies...
# вң“ Dependencies fetched
```

### koa pkg add

Add a new dependency to `Koa.toml`:

```bash
# Add with version
koa pkg add http@0.1.0 --git https://github.com/riipandi/koa-http

# Add with git URL and version separately
koa pkg add json --git https://github.com/riipandi/koa-json --version "^0.1.0"

# Add local path dependency
koa pkg add utils --path ./utils

# Add with branch
koa pkg add auth --git https://github.com/user/koa-auth --branch main
```

### koa pkg remove

Remove a dependency from `Koa.toml`:

```bash
koa pkg remove http
```

### koa pkg list

List installed dependencies:

```bash
koa pkg list

# Output:
# Dependencies (3)
#   http  v0.1.0  git+https://github.com/riipandi/koa-http
#   json  v0.1.0  git+https://github.com/riipandi/koa-json
#   utils v0.1.0  path+./utils
```

### koa pkg tree

Show dependency tree:

```bash
koa pkg tree

# Output:
# myapp v0.1.0
# в””в”Җв”Җ http v0.1.0
#     в””в”Җв”Җ json v0.1.0
```

### koa pkg outdated

Check for outdated dependencies:

```bash
koa pkg outdated

# Output:
# Checking for outdated dependencies...
# вң“ All dependencies are up to date
```

### koa pkg update

Update dependencies:

```bash
# Update all dependencies
koa pkg update

# Update specific dependency
koa pkg update http

# Output:
# в Ӣ Updating http...
# вң“ Dependencies updated
```

## Dependency Sources

### Git Repository

```toml
[dependencies]
http = { git = "https://github.com/riipandi/koa-http", version = "0.1.0" }
```

Options:
- `git` - Git repository URL
- `version` - Tag atau version (opsional)
- `branch` - Branch name (opsional, default: main/master)
- `rev` - Specific commit SHA (opsional)

### Local Path

```toml
[dependencies]
utils = { path = "./utils" }
```

### Standard Library

Stdlib is always available:

```toml
[dependencies]
std = { version = "0.1.0" }  # Virtual dependency
```

## Version Resolution

### Semantic Versioning

```toml
[dependencies]
http = { git = "https://...", version = "0.1.0" }   # Exact: 0.1.0
http = { git = "https://...", version = "^0.1.0" }  # Compatible: >=0.1.0, <0.2.0
http = { git = "https://...", version = "~0.1.0" }  # Patch: >=0.1.0, <0.2.0
http = { git = "https://...", version = "*" }       # Any: latest
```

### Branch/Commit

```toml
[dependencies]
http = { git = "https://...", branch = "main" }
http = { git = "https://...", rev = "a1b2c3d" }
```

## Using Dependencies

### Import External Packages

```typescript
// src/main.koa
import { HttpClient } from "http/client"
import { parse_json } from "json/parser"

fn main(): i32 {
    let client: HttpClient = HttpClient::new()
    let response: HttpResponse = await client.get("https://api.example.com")
    let data: Data = parse_json(response.body)
    0
}
```

### Dependency Entry Point

Dependencies must have a main file:

```
https://github.com/riipandi/koa-http
в”ңв”Җв”Җ Koa.toml        # Package metadata
в”ңв”Җв”Җ src/
в”Ӯ   в””в”Җв”Җ mod.koa        # Public exports
в””в”Җв”Җ examples/
```

`src/mod.koa`:
```typescript
// Re-export public items
pub { HttpClient, HttpResponse }
```

## Lockfile Details

### Purpose

Lockfile ensures reproducible builds:

```bash
# Developer A
koa build
# Generates Koa.lock dengan commit SHA

# Developer B (different time)
koa build
# Uses Koa.lock, same exact dependencies
```

### Format

TOML format (human-readable):

```toml
[[package]]
name = "http"
version = "0.1.0"
git = "https://github.com/riipandi/koa-http"
rev = "a1b2c3d4e5f6789..."
checksum = "sha256:abc123..."
```

JSON format (machine-readable, optional):

```json
{
  "packages": [
    {
      "name": "http",
      "version": "0.1.0",
      "git": "https://github.com/riipandi/koa-http",
      "rev": "a1b2c3d4e5f6789...",
      "checksum": "sha256:abc123..."
    }
  ]
}
```

### Checksums

SHA256 checksum dari source code:

```bash
# Checksum prevents tampering
checksum = "sha256:a1b2c3d4..."
```

## Workflows

### New Project

```bash
# Create new project
koa init myproject
cd myproject

# Add dependency
koa pkg add http --git https://github.com/riipandi/koa-http --version 0.1.0

# Fetch dependencies
koa pkg fetch

# Build
koa build
```

### Add Dependency

```bash
# Add dependency (auto-edits Koa.toml)
koa pkg add mydep --git https://github.com/user/mydep --version 1.0.0

# Fetch dependencies
koa pkg fetch

# Use in code
# import { Something } from "mydep/module"
```

### Remove Dependency

```bash
# Remove dependency (auto-edits Koa.toml)
koa pkg remove mydep
```

### Update Dependency

```bash
# Update specific dependency
koa update http

# Update all
koa update

# Review changes
git diff Koa.lock
```

## Virtual Registry

Tidak ada central registry, tapi gunakan conventions:

### GitHub Convention

```
https://github.com/{user}/koa-{name}
```

Examples:
- `https://github.com/riipandi/http`
- `https://github.com/riipandi/json`

### Git Tags

Gunakan Git tags untuk versioning:

```bash
git tag v0.1.0
git push --tags
```

Dependency resolution mencari tags:
- `v0.1.0` вҶ’ tag `v0.1.0`
- `0.1.0` вҶ’ tag `v0.1.0`

## Best Practices

### 1. Commit Koa.lock

```bash
git add Koa.lock
git commit -m "Add dependencies"
```

### 2. Use Specific Versions

```toml
# BAD: Branch only
http = { git = "...", branch = "main" }  # Changes unpredictably

# GOOD: Version constraint
http = { git = "...", version = "^0.1.0" }  # Predictable updates
```

### 3. Update Regularly

```bash
koa pkg update  # Get latest compatible versions
```

### 4. Review Changes

```bash
git diff Koa.lock  # Review what changed
```
